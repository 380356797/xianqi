# 卡通Q版象棋APP设计文档

## 一、项目概述
开发一款针对Android平台的卡通Q版风格中国象棋单机游戏，支持双人对战、单人对战（3个难度等级），包含音效、悔棋、计时、设置、胜率统计等功能。

## 二、技术选型
- **平台：** Android
- **开发语言：** Kotlin
- **UI框架：** Jetpack Compose (Material3)
- **架构模式：** MVVM
- **数据存储：** SharedPreferences
- **音效实现：** Android原生MediaPlayer
- **最低支持版本：** Android 7.0 (API 24)

## 三、功能清单

### 核心功能
1. **对战模式**
   - 双人对战：两人同屏对弈
   - 单人对战：与AI对战，3个难度等级
     - 简单难度：随机走棋 + 优先吃子
     - 中等难度：Alpha-Beta搜索2-3层 + 基础估值
     - 困难难度：Alpha-Beta搜索4-5层 + 优化估值

2. **游戏功能**
   - 悔棋功能：支持撤销上一步操作
   - 对局计时：显示双方用时
   - 胜负判断：自动识别将死、困毙、和棋
   - 走棋动画：棋子移动弹跳效果、吃子特效

3. **音效系统**
   - 背景音乐：循环播放轻松卡通风格背景音乐
   - 操作音效：走棋、吃子、将军、胜负对应音效
   - 音量调节：支持独立调节音乐和音效音量

4. **辅助功能**
   - 设置功能：音效开关、音量调节、悔棋次数设置
   - 胜率统计：各难度下的胜/负/和统计、总对局数

### 2. 核心类职责

#### ChessEngine（象棋规则引擎）
- 生成所有合法走法
- 验证走法是否合法
- 判断胜负状态（将死、困毙、和棋）
- 判断是否将军

#### AiPlayer（AI玩家）
- 实现不同难度的AI算法
- 根据当前棋盘状态计算最佳走法
- 难度等级通过搜索深度和估值函数复杂度区分

#### GameManager（游戏管理器）
- 管理游戏生命周期（准备、进行、暂停、结束）
- 管理回合切换
- 记录走棋历史
- 实现悔棋逻辑
- 管理计时功能

#### SoundManager（音效管理器）
- 初始化音效资源
- 提供播放/暂停/停止接口
- 管理音量设置

## 五、UI设计规范

### 视觉风格
- **整体风格：** 卡通Q版，明亮活泼
- **棋盘：** 暖色调木纹背景，格子线采用圆润设计
- **棋子：** 每个棋子都有独特的Q版卡通造型和表情
  - 红方：暖色调为主，棋子带有可爱表情
  - 黑方：冷色调为主，棋子带有酷帅表情
- **配色方案：**
  - 主色：暖橙色（#FF9800）
  - 辅助色：天蓝色（#2196F3）
  - 背景色：浅米色（#FFF8E1）
  - 棋盘色：深棕色木纹纹理
  - 红色棋子：#F44336
  - 黑色棋子：#212121

### 页面结构
1. **主菜单页面**
   - 大图标按钮：双人对战、单人对战、设置、统计
   - 可爱的游戏标题Logo
   - 背景点缀象棋元素小图标

2. **游戏页面**
   - 顶部：双方头像、昵称、计时、剩余悔棋次数
   - 中部：棋盘（居中自适应屏幕大小）
   - 底部：功能按钮（悔棋、暂停、认输）
   - 弹窗：胜负结果提示、设置弹窗

3. **设置页面**
   - 音量调节滑块（音乐、音效独立）
   - 开关选项（计时开关、音效开关）
   - 难度选择（单人模式默认难度）
   - 悔棋次数设置

4. **统计页面**
   - 总对局数、胜场、负场、和场统计
   - 各难度胜率柱状图
   - 最近对局记录列表

## 六、开发计划里程碑
1. **第一周：核心逻辑实现**
   - 象棋规则引擎开发
   - AI算法实现
   - 游戏管理器开发

2. **第二周：UI界面开发**
   - Q版主题设计
   - 棋盘、棋子组件开发
   - 所有页面UI实现
   - 动画效果实现

3. **第三周：功能完善与测试**
   - 音效系统集成
   - 数据存储实现
   - 功能联调
   - 性能优化与BUG修复

## 七、非功能需求
- **性能要求：** 启动时间<2秒，游戏过程无卡顿
- **适配要求：** 支持手机和平板，横竖屏自动适配
- **权限要求：** 不需要任何特殊权限
- **包体积要求：** <5MB
